# Dataset Workspace

Use `dataset/candidate/` for locally downloaded candidate datasets during
dataset selection and preprocessing research.

The `candidate/` directory is intentionally ignored by Git because it may contain
large third-party dataset files. Keep source URLs, licenses, and preprocessing
decisions in tracked docs or manifests before using a dataset for evaluation.

Cleaned evaluation inputs are generated by dataset-specific scripts:

```text
dataset/locomo/clean.py
dataset/locomo_interleaved/clean.py
dataset/eventqa/clean.py
dataset/mempal/clean.py
dataset/perma/clean.py
```

Each script writes the standard system interface into its local `processed/`
directory:

```text
memory_units.jsonl
query_records.jsonl
manifest.json
```

`processed/` directories are ignored because they are reproducible generated
data. Embedding caches are generated separately after model selection.

Build embeddings through an OpenAI-compatible vLLM embedding server with:

```bash
python dataset/build_embeddings.py --config configs/build_embeddings_template.json
```

The script supports batched concurrent requests and resumable output. Edit the
config's dataset paths, `base_url`, `model`, `batch_size`, and `concurrency`
before running a full build.

For multiple embedding models, keep one cache file per model, for example:

```text
dataset/locomo/processed/embeddings/bge-base-en-v1.5.json
dataset/locomo/processed/embeddings/e5-large-v2.json
```

Set `retrieval.embedding_cache_path` in the experiment config to the specific
model cache you want to evaluate.

`locomo_interleaved` reorders LoCoMO queries with conversation-level round-robin
interleaving while keeping memory units, query ids, and query text unchanged.
It reuses the original LoCoMO embedding cache.
