# Exp1 Evaluation Plan

## Baseline Parameters

Use this parameter set as the default high-pressure template unless a dataset
or ablation explicitly overrides it:

- retrieval: `embedding_retrieve`
- topk: `10`
- page.max_units: `1000`
- page.max_amplification: `5`
- renderer.width_scale: `4.0`
- foreground.min_coverage: `0.3`
- prewarm.min_candidate_coverage: `0.5`
- prewarm.max_new_in_old: `0.5`
- prewarm.max_inflight_pages: `1`
- runtime.foreground_max_tokens: `64`
- runtime.background_max_tokens: `1`
- runtime.background_chunk_tokens: `1024`
- page.max_visual_tokens: `60000` for unattended runs under a 64k context limit

## Mode Factor

These four modes are the main method comparison axis.

| Paper name | Config mode | Runtime behavior | Current status |
|---|---|---|---|
| baseline | `baseline` | Render current top-k as foreground only. Do not register pages and do not submit background pages. | Implemented |
| random | `random_anchor` | Retrieve top-k, then after fallback construct a random page under the amplification cap. No append foreground registration. | Implemented |
| temporal | `append` | Reactive temporal/path locality. Fallback top-k and warm-hit residuals are registered as foreground pages. | Implemented |
| semantic | `embedding_ball` with `radius_percentile=100` | Predictive semantic locality. Build nearest-neighbor pages under the amplification cap; warm-hit residuals can register foreground pages. | Implemented |

Expected interpretation:

- `baseline` measures pure foreground retrieval/rendering cost and answer quality.
- `random` controls for having cached pages of similar size without meaningful locality prediction.
- `temporal` tests reactive reuse from previously accessed units.
- `semantic` tests predictive reuse from embedding nearest-neighbor construction.
  Page size is controlled by `page.max_amplification`, not by a percentile cap.

## Main Experimental Axes

The first complete matrix should vary only mode, dataset, and pressure:

- mode: `baseline`, `random`, `temporal`, `semantic`
- pressure: LoCoMO uses `width_scale=1.0` and `width_scale=4.0`;
  EventQA/PERMA use `width_scale=1.0` for the first full run.
- dataset: LoCoMO, EventQA, PERMA

## Recommended Additional Main Controls

These are useful if time allows, but should not block the first full matrix:

- `semantic_ball_p5` as an ablation. This restores the old percentile-ball
  construction and measures prediction breadth versus read amplification.
- `temporal` and `semantic` with `max_amplification`: e.g. `3`, `5`, `8`.
  This measures how each method behaves under stricter page-size pressure.
- `max_inflight_pages`: `1` versus `5`.
  This separates locality quality from background prewarm throughput.
- `foreground.min_coverage`: `0.3` versus `0.5`.
  This tests how aggressive foreground page reuse should be.

## Notes

- Keep `foreground_max_tokens >= 64` for all quality-bearing foreground runs.
- `background_max_tokens=1` is acceptable because background requests are
  prefetch-only.
- For unattended runs, set `page.max_visual_tokens` or pass autoeval
  `--max-visual-tokens 60000`. Oversized warm-page foreground plans fall back
  to baseline top-k; oversized background pages are skipped.
- Session-level autoeval is the default for the full matrix.
- Use canonical config lists generated by
  `evaluation/autoeval/generate_exp1_configs.py`; the main list is
  `evaluation/autoeval/configs_exp1_main.txt`.
- Autoeval continues to the next session/config after retry exhaustion by
  default. Use `--stop-on-failure` only for fail-fast debugging.
- Each config writes `preflight.json` before execution. Hard preflight errors
  skip that config; warnings are retained for later audit.
